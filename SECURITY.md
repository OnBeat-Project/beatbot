# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | ✅ Yes             |
| 5.0.x   | ❌ No              |
| 4.0.x   | ✅ Yes             |
| < 4.0   | ❌ No              |

Only the versions marked as supported will receive security fixes. Please make sure you are using a supported version to stay protected.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

- **Do not** open a public issue for security-related bugs.
- Send a detailed report describing the vulnerability, steps to reproduce it, and its potential impact.
- Include relevant logs, screenshots, or proof of concept if possible.

After reporting:
- You can expect an initial response within **48–72 hours**.
- If the vulnerability is accepted, we will work on a fix and notify you once it’s resolved.
- If the report is declined, we will explain the reason clearly.

We appreciate responsible disclosures and your help in keeping the project secure.
